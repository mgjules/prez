What is Inertia.js?
---

**Inertia.js** = Modern SPAs without API complexity

## Key Features
- **No APIs needed** - Direct data passing between server and frontend
- **SPA experience** - Smooth navigation without full page reloads
- **Framework agnostic** - Works with React, Vue, Svelte, Angular
- **Server-side routing** - Traditional web routing with modern UX
- **Asset versioning** - Automatic cache busting and updates
- **Partial reloads** - Request only the data you need
- **History management** - Browser back/forward works naturally

## How It Works
- Server renders page components with data
- Client receives JSON page objects via XHR
- Frontend framework updates components without page reload
- Seamless SPA experience with traditional web development

<!-- end_slide -->

The Inertia.js Protocol
---

### How It Works
- **First visit**: Full HTML response with assets and initial page data
- **Subsequent visits**: XHR requests with JSON responses
- **No APIs**: Data passed directly between server and frontend

### Request Flow
1. **Initial Request** → Full HTML page with `data-page` attribute
2. **Navigation** → XHR with `X-Inertia: true` header
3. **Response** → JSON page object with component, props, URL, version
4. **Render** → Client-side component updates without full page reload

### Key Headers
- `X-Inertia: true` - Identifies Inertia requests
- `X-Inertia-Version` - Asset version checking
- `Vary: X-Inertia` - Response caching control

### Page Object Structure
```json +line_numbers +no_background
{
    "component": "User/Index",
    "props": { "users": [...] },
    "url": "/users",
    "version": "abc123"
}
```

Inertia.js vs HTMX
---

## Technical Comparison

| Aspect | Inertia.js | HTMX |
|--------|-------------|-------|
| **Frontend** | React/Vue/Svelte components | HTML with attributes |
| **State Management** | Client-side (Redux, Pinia, etc.) | Server-side |
| **Bundle Size** | Larger (framework + components) | Smaller (HTMX only) |
| **Learning Curve** | Steeper (framework knowledge) | Gentler (HTML attributes) |
| **SEO** | Requires SSR setup | Excellent out of box |
| **Mobile Performance** | Excellent (SPA experience) | Good (page reloads) |
| **Team Skills** | Frontend developers | Backend developers |

Quick Start

## Installation

```bash +line_numbers +no_background
go get github.com/romsar/gonertia/v2
```

## Basic Setup

```go +line_numbers +no_background
package main

import (
    "log"
    "net/http"
    inertia "github.com/romsar/gonertia/v2"
)

func main() {
    i, err := inertia.New(`
        <!DOCTYPE html>
        <html>
        <head>
            <meta charset="utf-8">
            {{ .inertiaHead }}
        </head>
        <body>
            {{ .inertia }}
            <script src="/app.js"></script>
        </body>
        </html>
    `)
    if err != nil {
        log.Fatal(err)
    }
    
    mux := http.NewServeMux()
    mux.Handle("/", i.Middleware(homeHandler(i)))
    
    log.Println("Server starting on :8080")
    http.ListenAndServe(":8080", mux)
}
```

Route Handlers

## Go Handler

```go +line_numbers +no_background
func homeHandler(i *inertia.Inertia) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        err := i.Render(w, r, "Home", inertia.Props{
            "name": "World",
            "time": time.Now().Format("2006-01-02 15:04:05"),
        })
        if err != nil {
            http.Error(w, err.Error(), http.StatusInternalServerError)
            return
        }
    })
}
```

Frontend Component
---

## Svelte Example

```svelte +line_numbers +no_background
<script>
  import { inertia } from "@inertiajs/svelte";

  export let users;

  function formatDate(date) {
    return new Date(date).toLocaleDateString();
  }
</script>

<div>
  <h1>Users</h1>
  {#each users as user}
    <div>
      <a use:inertia href="/users/{user.id}">{user.name}</a>
      <p>{user.email}</p>
      <small>Created: {formatDate(user.created_at)}</small>
    </div>
  {/each}
  <a use:inertia href="/users/create">Create New User</a>
</div>
```

## Key Points
- Use `@inertiajs/svelte` for reactive components
- Props passed directly from Go handlers
- No API calls needed
