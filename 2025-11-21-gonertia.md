What is Gonertia?
---

**Gonertia** = Go adapter for Inertia.js

**Inertia.js** = Modern SPAs without API complexity

## The Problem
- Traditional SPAs need separate frontend + backend
- APIs to design, build, and maintain
- Duplicate validation and auth logic
- Complex deployment workflows

## The Solution
- Single Go application with modern frontend
- No API layer - data passed directly
- Server-side routing with SPA experience
- Deploy one binary

Quick Start

## Installation

```bash +line_numbers +no_background
go get github.com/romsar/gonertia/v2
```

## Basic Setup

```go +line_numbers +no_background
package main

import (
    "log"
    "net/http"
    inertia "github.com/romsar/gonertia/v2"
)

func main() {
    i, err := inertia.New(`
        <!DOCTYPE html>
        <html>
        <head>
            <meta charset="utf-8">
            {{ .inertiaHead }}
        </head>
        <body>
            {{ .inertia }}
            <script src="/app.js"></script>
        </body>
        </html>
    `)
    if err != nil {
        log.Fatal(err)
    }
    
    mux := http.NewServeMux()
    mux.Handle("/", i.Middleware(homeHandler(i)))
    
    log.Println("Server starting on :8080")
    http.ListenAndServe(":8080", mux)
}
```

Route Handlers

## Simple Handler

```go +line_numbers +no_background
func homeHandler(i *inertia.Inertia) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        err := i.Render(w, r, "Home", inertia.Props{
            "name": "World",
            "time": time.Now().Format("2006-01-02 15:04:05"),
        })
        if err != nil {
            http.Error(w, err.Error(), http.StatusInternalServerError)
            return
        }
    })
}

func usersHandler(i *inertia.Inertia) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        users := []User{
            {ID: 1, Name: "Alice", Email: "alice@example.com"},
            {ID: 2, Name: "Bob", Email: "bob@example.com"},
        }
        
        err := i.Render(w, r, "Users", inertia.Props{"users": users})
        if err != nil {
            http.Error(w, err.Error(), http.StatusInternalServerError)
            return
        }
    })
}
```

Forms & Validation
---

## Form Handler

```go +line_numbers +no_background
func createUserHandler(i *inertia.Inertia) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        if r.Method == "GET" {
            i.Render(w, r, "Users/Create", inertia.Props{})
            return
        }
        
        if err := r.ParseForm(); err != nil {
            ctx := inertia.SetValidationErrors(r.Context(), inertia.ValidationErrors{
                "name": "Name is required",
                "email": "Invalid email format",
            })
            i.Back(w, r.WithContext(ctx))
            return
        }
        
        user := User{
            Name:  r.FormValue("name"),
            Email: r.FormValue("email"),
        }
        
        i.Redirect(w, r, "/users")
    })
}
```

Advanced Features
---

## Dynamic Props

```go +line_numbers +no_background
// Optional props (lazy loading)
props := inertia.Props{
    "user": inertia.Optional(func() (any, error) {
        return getCurrentUser(r), nil
    }),
}

// Always included props
props := inertia.Props{
    "auth": inertia.Always(map[string]any{
        "user": getCurrentUser(r),
    }),
}

// Deferred props (loaded after initial render)
props := inertia.Props{
    "reports": inertia.Defer(func() (any, error) {
        return generateReports(), nil
    }),
}
```

When to Use Gonertia
---

## Perfect For
- **Internal tools** and admin dashboards
- **SaaS applications** with authenticated users
- **Content management systems**
- **Rapid prototyping** and MVPs
- **Go teams** wanting modern frontends

## Not Ideal For
- **Public APIs** consumed by third parties
- **Mobile apps** requiring separate backend
- **Microservices** with separate frontend teams

Benefits
---

## Development Benefits
- **Single codebase** - No separate frontend/backend repos
- **Type safety** - Shared structs between Go and frontend
- **Simplified deployment** - One binary to deploy
- **Reduced complexity** - No API design/maintenance
- **Better performance** - Direct data passing

## Team Benefits
- **Go developers stay productive** - Use familiar patterns
- **Full-stack Go** - End-to-end type safety
- **Simplified onboarding** - One stack to learn
- **Easier debugging** - Single process to trace
